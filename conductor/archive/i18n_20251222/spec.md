# Spec: 完善国际化（i18n）支持

## 1. 目标

此轨道的目标是全面审查和完善项目的国际化（i18n）支持。虽然项目中已存在基本的i18n设置，但需要确保所有面向用户的文本都已正确地从Vue组件中提取，并通过i18n机制进行管理。这将使用户能够根据其语言偏好，在不同语言（如英语和简体中文）之间无缝切换。

## 2. 功能需求

-   **文本外部化**：
    -   审查所有Vue组件，识别所有硬编码的面向用户的文本字符串。
    -   将所有识别出的文本字符串替换为对i18n key的引用。
    -   在 `src/locales/en.json` 和 `src/locales/zh-CN.json` 文件中为这些key添加相应的翻译。
-   **组件集成**：
    -   确保所有组件（包括按钮、标签、对话框、菜单项等）都能正确显示翻译后的文本。
    -   验证动态内容（如错误消息、成功提示、确认对话框等）也能通过i18n进行翻译。
-   **语言切换**：
    -   （如果尚未实现）提供一个用户界面元素（如下拉菜单或切换按钮），允许用户在支持的语言之间进行切换。
    -   确保语言切换后，整个应用界面的文本会立即更新，无需重新加载页面。

## 3. 非功能性需求

-   **可维护性**：i18n的实现应易于维护和扩展。添加新的语言或修改现有翻译应尽可能简单。
-   **性能**：语言包的加载和切换不应对应用的性能产生显著影响。

## 4. 验收标准

-   应用中所有面向用户的文本都已从源代码中移除，并替换为i18n key。
-   `en.json` 和 `zh-CN.json` 文件包含了所有UI文本的完整翻译。
-   用户可以在不刷新应用的情况下，在英语和简体中文之间切换，并且所有文本都会相应地更新。
-   在添加或修改翻译时，无需对组件代码进行更改。
